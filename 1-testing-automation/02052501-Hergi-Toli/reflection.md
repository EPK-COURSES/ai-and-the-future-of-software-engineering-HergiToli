\# Personal Reflection



Before doing this research, I thought AI was mainly useful for writing code faster. After reading about AI testing, debugging, and automation, I understood that AI can also help developers find errors, create test cases, and save time during software development.



What surprised me most is that AI can help with testing, not only with coding. Testing is very important because software can look correct but still have hidden bugs. AI can make this process faster, but it cannot completely replace the developer.



My biggest concern is that some developers may trust AI too much. If a developer accepts AI suggestions without understanding them, the software may still contain mistakes or security problems. This is why human supervision is still very important.



I think AI creates good opportunities for software engineers. It can help with repetitive tasks, explain errors, and support debugging. This gives developers more time to focus on understanding the problem and improving the quality of the software.



In the future, I think software engineers should improve their testing skills, debugging skills, critical thinking, and ability to use AI tools responsibly. Personally, I will try to use AI as an assistant, but I will not depend on it completely. I want to understand the code and the tests before accepting any AI-generated result.
